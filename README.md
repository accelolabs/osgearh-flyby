# Что это?
Это консольный инструмент на базе osgEarth для формирования изображения камеры БПЛА по географическим координатам (altitude, latitude, longitude) и ориентационным угла (yaw, pitch, roll) с выводом результата в оперативную память или графический файл.

# Процесс сборки
Проект зависит от библиотеки [osgEarth](https://github.com/pelicanmapping/osgearth). Чтобы этот проект заработал, нужно собрать osgEarth вручную и положить рядом с корневой директорией этого проекта (`../osgearth/build`).

После этого проект может быть собран из своей корневой директории:
```
mkdir build && cd build
cmake ..
cmake --build . --config Release
```

# Использование
Используйте команду `--help`, чтобы посмотреть доступные опции. Некоторые из них обязательны. Для работы программы обязателен `.earth` файл, есть `example/osm.earth` для тестирования работы программы.

Из-за особенностей рендеринга osgEarth существует параметр `--render-seconds`, нужный, чтобы osgEarth успел отрисовать изображение. Его значение по умолчанию составляет 30 секунд.

# Сохранение в ОЗУ
Сохранение в ОЗУ работает исключительно на Linux, так как сохраняет изображение в `/dev/shm`. 

```
cat /dev/shm/OSG_Capture_Memory | magick -depth 8 -size 512x512 rgba:- -flip output_from_ram.jpg
```
